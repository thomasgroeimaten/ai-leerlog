## 💡 Belangrijkste lessen

### IndyDevDan — "Building Multi-Agent Systems With Finite State Machines // Applied AI Meetups July 2026"

- **FSM = deterministisch skelet voor agent-coördinatie**: een Finite State Machine definieert welke agent verantwoordelijk is per fase en precies onder welke condities de workflow naar de volgende staat gaat. Geen vage "de agent beslist zelf" — de overgangen zijn voorspelbaar en auditeerbaar
- **Actor Model als complement**: elke agent is een actor die berichten asynchroon verwerkt zonder gedeelde mutable state. Dit maakt multi-agent workflows horizontaal schaalbaar: agent A en agent B kunnen parallel lopen zonder race conditions
- **"De wereld is state machines all the way down"**: elke zakelijke workflow — leadkwalificatie, offerteproces, klanttraject — is al een toestandsmachine. Het expliciet modelleren als FSM maakt die structuur zichtbaar en beheerbaar voor agents
- **2026 = jaar van multi-agent productie**: de infrastructuur voor gecoördineerde agents (tools, SDKs, orchestratieharnessen) is volwassen genoeg voor productiedeployment; bedrijven die nu FSM-architecturen implementeren bouwen een duurzame voorsprong op
- **MetaAgent-bewijs**: academisch werk (ICML 2025) toont aan dat FSM-gebaseerde multi-agent systemen automatisch construeerbaar zijn vanuit een taakomschrijving — de patronen die IndyDevDan bespreekt zijn niet theoretisch maar al gedistilleerd in werkende frameworks

Bron: [Building Multi-Agent Systems With Finite State Machines — IndyDevDan](https://www.youtube.com/watch?v=Onq9nHMN0oQ)

## 🔧 Concreet toepasbaar bij Groeimaten

**FSM-architectuur voor de leadkwalificatie-workflow**
Groeimaten's leadkwalificatie voor keuken-/badkamerbedrijven is al een toestandsmachine — alleen impliciet. Maak hem expliciet:

Toestanden: `NIEUW_LEAD → EERSTE_CONTACT → GEKWALIFICEERD → AFSPRAAK_GEPLAND → GEWONNEN / VERLOREN / SLAPEND`

Transitiecondities (bepaal deze per klant):
- `NIEUW_LEAD → EERSTE_CONTACT`: trigger = formulierinzending of websitebezoek >3 pagina's
- `EERSTE_CONTACT → GEKWALIFICEERD`: conditie = lead antwoordt binnen 24u + budget bevestigd
- `GEKWALIFICEERD → AFSPRAAK`: conditie = datum gekozen in Calendly of eigen planner

Elke agent (leadkwalificatie-agent, e-mailschrijf-agent, CRM-update-agent) krijgt dan een duidelijke verantwoordelijkheidszone binnen de FSM. Dit maakt het systeem auditable — je kunt precies zien in welke staat elke lead zit, welke agent hem vasthoudt, en waarom een transitie nog niet plaatsvond.

**Praktische eerste stap**: schets de FSM op papier voor één klant, vertaal naar een CLAUDE.md met de toestanden en transitieregels, en laat een agent de eerste workflow testen.

**Fable 5 benchmarken op klantwerk**
Nu Fable 5 terug is binnen de weeklimiet: neem één complexe klantrapportage die je eerder met Opus 4.8 hebt gemaakt en genereer hem opnieuw met Fable 5. Fable is getraind op een groter capabiliteitenspectrum — met name voor redenering over meerdere stappen. Dit is direct relevant voor proposals, keuzebegeleiding en renovatieplanning.

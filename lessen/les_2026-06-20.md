## 🚨 KRITIEK — nu uitvoeren

### 1. Bouw vandaag je eerste agent loop voor een terugkerende Groeimaten-taak
**WAT:** Kies één terugkerende taak (bijv. maandelijkse SEO-rapportage per klant, of wekelijkse advertentiekopij). Maak een map `loops/[taaknaam]/` aan met drie bestanden: `VISION.md` (wat moet eruit komen + wanneer is het klaar?), `CLAUDE.md` (de vaste regels: format, stijl, verboden), `PROMPT.md` (de kernopdracht). Voeg drie harde begrenzingen toe aan CLAUDE.md: max 20 iteraties, stop bij herhaalde fout, tokenbudget.
**WAAROM:** Dit is de concrete omschakeling van "elke dag prompts tikken" naar "één keer bouwen, daarna autonoom draaien". Boris Cherny (creator Claude Code) en Nate Herk beschrijven dit als de grootste productiviteitssprong die beschikbaar is. Elke dag uitstel = handmatig werk dat Claude kan doen.
**Eerste stap:** Open Claude Code → maak `loops/seo-rapportage/VISION.md` → schrijf 10 regels: wat is het doel, wat moet het rapport bevatten, hoe ziet "klaar" eruit. Klaar. Morgen de volgende twee bestanden.

### 2. Installeer claude-mem voor persistent geheugen over al je tools
**WAT:** `claude-mem` (open-source, `github.com/thedotmack/claude-mem`) is een MCP-server die je AI-context persistent maakt en deelt tussen Claude Code, Cursor, Codex en andere tools.
**WAAROM:** Op dit moment verlies je al je klantcontext zodra je een nieuwe chat start of naar een andere tool gaat. Met claude-mem herinnert Claude je klanten, tone-of-voice, architectuurkeuzes en eerdere beslissingen automatisch — over sessies en tools heen. Dit is structurele tijdwinst, niet een nice-to-have.
**Eerste stap:** Ga naar `cmem.ai`, maak gratis account aan, voeg de MCP-link toe aan je Claude Code configuratie (`~/.claude/settings.json` → `mcpServers`). Kost 10 minuten.

## 💡 Belangrijkste lessen (vooral Claude)

### Nate Herk — "Finally. Agent Loops Clearly Explained."
- Een agent loop heeft één vast patroon: **prompt → output lezen → klaar? → herhaal of stop** — draait automatisch op timer of cron, geen handmatige tussenkomst nodig
- De drie ankerbestanden zorgen voor stabiliteit over sessies heen: `VISION.md` = doel + klaar-criteria, `CLAUDE.md` = operationele regels per iteratie, `PROMPT.md` = de kernopdracht
- Elke herhaalde fout die je corrigeert moet in `CLAUDE.md` vastgelegd worden — zo wordt elke volgende run intelligenter dan de vorige; fouten worden institutioneel geheugen
- Bouw altijd drie harde begrenzingen in: **max iteratieaantal** (bijv. 20), **no-progress detectie** (stop bij herhaalde identieke fouten), **tokenbudget ceiling**
- "Verificatie" in een loop betekent de output écht testen — Cat Wu (Claude Code): de app opstarten, door de UI klikken, niet alleen een lint-check uitvoeren

Bron: [Finally. Agent Loops Clearly Explained. — Nate Herk](https://www.youtube.com/watch?v=EuzYhzB0vbI)

## 🔧 Concreet toepasbaar bij Groeimaten

**Loop voor maandelijkse klantrapportages**
Kies één klant als pilot. Maak `loops/rapportage-[klantnaam]/` met de drie ankerbestanden. VISION.md: wat bevat het rapport, hoe ziet "klaar" eruit. CLAUDE.md: stijlgids, vaste rubrieken, verboden fouten uit vorige rapportages. PROMPT.md: haal campagnedata op, analyseer, schrijf rapport. Loop triggert automatisch op de eerste maandag van de maand via Claude Code.

**Fout = les: CLAUDE.md updaten na elke sessie**
Elke keer dat Claude een fout maakt in klantwerk (foute toon, verkeerd format, vergeten context) → schrijf de correctie direct in de klant-CLAUDE.md. Niet in je hoofd, niet in een sticky note. Zo wordt elke volgende run beter dan de vorige — dit is het werkende principe achter loop engineering.

**claude-mem per klantproject**
Na installatie: maak per klant een geheugenlaag aan met USP's, tone-of-voice, doelgroep en campagne-learnings. Dit geheugen werkt dan over alle sessies heen — geen context-verlies meer. Relevant voor badkamer- en keukenbedrijven met veel productcategorieën en seizoensgebonden communicatie.

**Model-strategie voor bulk-content**
Voor tegelshowrooms of keukendealer-catalogi met 50+ producten: gebruik GLM 5.2 (gratis) voor bulk-drafts van productteksten en SEO-beschrijvingen, laat Claude Opus 4.8 de eindreview doen. Significant goedkoper dan alles op Opus draaien.

**Kimi K2.7 op de radar**
Als Cole Medin een full-video uitbrengt met benchmarks, evalueer dan of Kimi K2.7 GLM 5.2 kan vervangen als gratis bouw-model in de agent-stack.

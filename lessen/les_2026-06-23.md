## 🚨 KRITIEK — nu uitvoeren

### Test git worktrees voor parallelle Claude Code loops (15 minuten)
**WAT:** Maak een git worktree aan voor je ai-leerlog repo en draai twee Claude Code sessies tegelijk in aparte terminalvensters — één voor ai-leerlog, één voor een andere loop.

**WAAROM:** IndyDevDan legt uit dat sequentieel draaien van Claude Code een toolbeperking is, geen taakbeperking. Als je kick-off loop en ai-leerlog loop allebei 's ochtends starten, wacht de tweede nu op de eerste. Worktrees halen dit weg in twee commando's.

**Eerste stap (in repo-root):**
```bash
git worktree add ../ai-leerlog-worker main
```
Open vervolgens een tweede terminal in `../ai-leerlog-worker` en start Claude Code daar. Beide sessies draaien nu parallel zonder bestandsconflicten.

## 💡 Belangrijkste lessen

### IndyDevDan — "Git Worktrees Explained — Run Multiple AI Agents in Parallel" *(7d-fallback, 16 juni 2026)*
- Meeste developers draaien Claude Code **sequentieel** — één taak klaar, dan pas de volgende; dit is een toolbeperking, geen taakbeperking
- **Git worktrees** geven elke agent een eigen geïsoleerde working directory en git index; alle sessies delen wel dezelfde object store (commits, blobs)
- Resultaat: geen file-level conflicten, geen context-contaminatie tussen sessies, geen lock-contention bij meerdere parallelle Claude Code-runs op dezelfde repo
- Gemeten voordeel: **3× hogere doorvoer** bij parallelle agent-sessies versus sequentieel werken
- Claude Code heeft native worktree-ondersteuning — docs: [code.claude.com/docs/en/worktrees](https://code.claude.com/docs/en/worktrees)
- Basiscommando: `git worktree add <pad> <branchnaam>` → start Claude Code direct in de nieuwe directory
- Praktische toepassing: meerdere loops tegelijk draaien — elke loop in eigen worktree, geen interferentie

## 🔧 Concreet toepasbaar bij Groeimaten

**Parallelle loops via git worktrees**
Heb je meerdere Claude Code loops (ai-leerlog, kick-off, klant-rapportage, social content generator)? Ze hoeven niet meer te wachten. Worktrees geven elke loop een eigen werkruimte in dezelfde repo. Als elke loop gemiddeld 5 minuten duurt en je hebt er 4, leveren parallelle worktrees 15 minuten tijdwinst per ochtend op.

**Kanttekening**
Worktrees lonen het meest bij zware, gelijktijdige loops. Voor een lichte dagelijkse routine is één-per-keer prima. Begin met twee parallel en meet of de snelheidswinst de extra complexiteit rechtvaardigt.

## 💡 Belangrijkste lessen (vooral Claude)

### Nate Herk — "How to Build Effective Claude Code Agents in 2026"
- Stop met "prompt and pray": stuur je agents actief aan via een planning-fase gevolgd door een verificatieloop — niet één lange chat
- Elk model heeft een **"dumb zone"**: zodra de context te groot wordt, stijgen fouten sterk; detecteer dit moment en start een nieuwe sessie
- Chain meerdere korte agent-sessies voor grote taken i.p.v. alles in één lange run te stoppen — dit voorkomt halverwege-falen
- Bouw verificatiestappen in: laat de agent zijn eigen output reviewen voor je verder gaat

### Cole Medin — "The Creators of Claude Code and OpenClaw don't Prompt Their Agents Anymore?!"
- De top-engineers schrijven nauwelijks nog prompts — ze bouwen een **self-prompting loop** (harness) die voor hen opdrachten genereert en uren autonoom runt
- Het principe: geef de agent een **specificatie** (WHAT + constraints), niet een opdracht (DO THIS) — de loop regelt de rest
- OpenClaw en Claude Code's makers delen exact dezelfde filosofie: harness first, model second
- Gevolg: minder tijd kwijt aan dagelijks prompten, meer output per uur

Bron: [The Creators of Claude Code and OpenClaw don't Prompt Their Agents Anymore?! — Cole Medin](https://www.youtube.com/watch?v=UztrFXaSWv0)

## 🔧 Concreet toepasbaar bij Groeimaten

**Second Brain per klant (doen: deze week)**
Maak een `CLAUDE.md` per klant met: bedrijfsnaam, producten, tone of voice, USP's, doelgroep, eerdere campagne-resultaten. Dit is niveau 2 van Nate's Second Brain. Elke Claude-sessie voor die klant laadt automatisch de context — geen copy-pasten meer.

**Agent-loop voor terugkerende taken**
Voor maandelijkse deliverables (SEO-rapportage, social posts, advertentiekopij) bouw je één self-prompting loop: geef de agent een specificatie-bestand, laat hem draaien. Exact de aanpak van Cole Medin — minder handmatig sturen, meer output.

**GLM 5.2 voor bulk-content**
Bij bulk-productteksten (keukendealers met 50+ producten, tegelshowroom met catalogus) test GLM 5.2 als goedkoper model via Claude Code. Gebruik Claude Sonnet 4.6 voor eindreview en kwaliteitscheck.

**Fable 5-fallback: nu regelen**
Controleer vandaag alle actieve agent-configs op modelkeuze. Switch Fable 5 → Opus 4.8.

**Omnigent: op de radar voor Q3**
Wanneer Groeimaten meerdere agents runt (content agent, SEO agent, ads agent, klant-rapportage agent), is Omnigent gratis en open-source de juiste coördinatielaag. Begin nu met de GitHub-repo bekijken.

## 🚨 KRITIEK — nu uitvoeren

### Verwijder de BASH-tool uit je Claude Code agent harnesses (10 minuten)

**WAT:** Open je Claude Code-configuratie en controleer welke tools actief zijn in je agent harnesses.

**WAAROM:** IndyDevDan toont aan dat de BASH-tool in Claude Code en andere agent harnesses een directe beveiligingsrisico is: het geeft je agent onbeperkte shell-toegang. Eén kwaadaardige instructie in externe input (een website die je agent bezoekt, een API-response, een ingelezen bestand) kan via BASH destructieve commando's uitvoeren op je systeem. Zeker relevant als jij of je team Claude Code gebruikt in workflows die externe data verwerken.

**Actie:**
1. Ga naar je Claude Code-instellingen (CLAUDE.md of settings.json)
2. Controleer of de `Bash`-tool actief is in je agent-loops
3. Als je de Bash-tool niet strikt nodig hebt: schakel hem uit; gebruik in plaats daarvan `Read`, `Write`, `Edit`, `Glob`, `Grep`
4. Als je BASH wel nodig hebt: beperk het tot een whitelist van toegestane commando's

## 💡 Belangrijkste lessen

### Cole Medin — "Context Engineering 101: The Simple Strategy to 100× AI Coding"

- Vibe coding (improviseren per prompt) geeft inconsistente output; **context engineering** is de stap daarna
- **PRPs (Product Requirements Prompts)** = herbruikbare context-documenten die de AI voor elke taak alles geven wat die nodig heeft: architectuurbeslissingen, projectregels, best practices, codevoorbeelden, validatiestappen
- Bij vibe coding is kwaliteit prompt-afhankelijk; bij context engineering is kwaliteit **systeem-afhankelijk** — je bouwt het één keer, het werkt altijd
- PRPs werken met Claude Code, maar ook met elke andere AI coding assistant (Cursor, Gemini Code Assist, Copilot)
- Cole sprak op 24 juni op Accelerate Chicago over "Agentic Engineering With Context Driven Development" — context engineering is de standaard die production-grade AI-development onderscheidt van one-off experimenten

### IndyDevDan — "Engineers, DELETE the BASH Tool: Agentic Security For Pi Agent and Claude Code"

- De **Bash-tool** in Claude Code geeft agents directe shell-toegang — de meest gevaarlijke tool in je harness
- Aanvalsvector (prompt injection via Bash): agent leest externe bron → externe bron bevat kwaadaardige instructie → agent voert `rm -rf`, `curl`, of data-exfiltratie uit via Bash
- Praktijktest: 95% van engineers die Bash actief laten in hun agent harness zijn kwetsbaar voor deze aanval
- **Alternatief:** gebruik gerichte, scoped tools: `Read` (lees bestanden), `Write`/`Edit` (schrijf bestanden), `Glob` (zoek bestanden), `Grep` (zoek inhoud) — die hebben geen onbeperkte shell-toegang
- Pi Agent als benchmark: bewust gebouwd zonder Bash-tool → kleinere attack surface → veiliger bij externe data-verwerking

Bron: [Engineers, DELETE the BASH Tool: Agentic Security For Pi Agent and Claude Code — IndyDevDan](https://www.youtube.com/watch?v=yBcmIoA-vGs)

## 🔧 Concreet toepasbaar bij Groeimaten

**UGC-video's voor keuken- en badkamer-klanten via n8n**
De Nate Herk-workflow sluit perfect aan op wat keuken- en tegelbedrijven nodig hebben: één productfoto van een keuken, tegel of badkamer-concept → automatisch gegenereerde UGC-stijl video voor social media. Kosten: €0,15 per clip. Groeimaten kan dit aanbieden als onderdeel van een content-as-a-service pakket. Start met de gratis n8n-template en vervang Sora/Veo door wat beschikbaar is in jouw stack. ROI: klantvideo-productie gaat van €300 (filmdag) naar €0,15 per clip.

**Bouw een Groeimaten-PRP-template voor Claude Code**
Maak één markdown-document met alle vaste Groeimaten-context: branche (keukens/badkamers/tegels), tone of voice, SEO-conventies voor Nederlandse markt, klantstructuur (MKB-installateurs), en veelgebruikte tools. Laad dit als context vóór elke Claude Code-sessie. Dit vervangt 10 minuten uitleg bij elke nieuwe sessie en zorgt voor consistente output. Sla het op als `groeimaten-context.md` of `CLAUDE.md` in elk klantproject.

**Systeem-architectuur vóór tools — Jake Van Clief's principe**
Voordat je een nieuwe AI-tool of agent bouwt voor een klant: definieer eerst de instructies (wat doet het?), de context (welke kennis heeft het nodig?) en het geheugen (wat onthoudt het tussen runs?). Dat klinkt simpel maar voorkomt 80% van de problemen. Bouw per klant-dienst een eigen map met die drie bestanden. Je kunt dit opschalen zonder platform-lock-in.

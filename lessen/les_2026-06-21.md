## 💡 Belangrijkste lessen

### Cole Medin — "The Creators of Claude Code and OpenClaw don't Prompt Their Agents Anymore?!"
- De beste agentic engineers tikken geen losse prompts meer: ze **bouwen loops die voor hen prompten** en laten die uren draaien — handmatig prompten is de inefficiënte manier van werken
- Boris Cherny (Claude Code-creator) + OpenClaw-creator werken **harness-first**: hun werk zit in configuratiebestanden en geheugenlagen, niet in ad-hoc prompts
- **OpenClaw-patroon**: `SOUL.md` = karakter + persistente context, `user.md` = wie ben ik + wat wil ik — beide worden vóór elke sessie automatisch geïnjecteerd; rechtstreeks toepasbaar in Claude Code
- Als je jezelf betrapt op het herschrijven van dezelfde prompt → dat is het signaal: bouw er een loop van
- Praktisch onderscheid: de **harness** (VISION.md + CLAUDE.md + PROMPT.md) is het product; de prompt is alleen de aansturing

Bron: [The Creators of Claude Code and OpenClaw don't Prompt Their Agents Anymore?! — Cole Medin](https://www.youtube.com/watch?v=UztrFXaSWv0)

### Nate Herk — "How to Build Effective Claude Code Agents in 2026"
- **Prompt and pray vs direct and verify**: het verschil tussen vibe coding en echte resultaten zit in een expliciete planning + verificatiestap vóór en ná elke agent-sessie
- Elk model heeft een **"dumb zone"** — een punt waarop het evidente dingen begint te missen door context-drift; herken dit vroeg, bouw een checkpoint vóórdat je die grens bereikt
- **Ketening van sessies**: knip grote taken bewust in deeltaken met overdraag-context; één lange sessie tot het einde doortrekken is het meest voorkomende faalpatroon
- Plan-fase is niet optioneel: laat de agent eerst een plan schrijven, geef goedkeuring, dan pas executie — dit filtert de meeste fouten al in de eerste stap weg

Bron: [How to Build Effective Claude Code Agents in 2026 — Nate Herk](https://www.youtube.com/watch?v=RzLV8sfFdMM)

## 🔧 Concreet toepasbaar bij Groeimaten

**Harness-first: stop met losse prompts voor terugkerende taken**
Identificeer de 3 meest herhaalde prompt-taken van deze week (bijv. advertentiekopij draft, SEO-meta schrijven, klantrapport). Die worden loops. Gebruik de structuur van gisteren: VISION.md + CLAUDE.md + PROMPT.md. Na vandaag mag je die prompts nooit meer handmatig tikken.

**Checkpoint na elke agent-sessie**
Voeg in je bestaande loops een expliciete verificatiestap toe: laat Claude na elke major output de done-criteria in VISION.md aflopen. Dit is de dumb-zone-detectie in de praktijk — een checkpoint van 5 regels voorkomt 30 minuten terugwerken.

**Model-strategie voor Groeimaten-workflows**
- Campagnestrategie / copywriting-aanpak plannen → `claude-opus-4-8`
- Bulk-executie (50 productteksten, tegelcatalogus, SEO-beschrijvingen) → GLM 5.2 of Kimi K2.7
- Review + kwaliteitscheck → `claude-sonnet-4-6` of Opus
- Als een model wegvalt: één laag verwisselen, de harness draait gewoon door

**OpenClaw SOUL.md-patroon voor klantprojecten**
Per klant: maak een `SOUL.md` met: merkpersoonlijkheid, tone-of-voice, doelgroep, verboden woorden, USP's. Voeg toe aan de klant-loop-CLAUDE.md als vaste injectie. Klanten met veel productcategorieën (tegels, keukens) profiteren direct: geen context-verlies meer bij nieuwe sessies.

**AI-visuals testen voor showrooms**
Test A/B AI-gegenereerde lifestyle-beelden (keuken/badkamer) tegen stockfoto's in Meta-campagnes voor keukenbedrijven. Meten wat converteert — de authenticiteits-discussie is pas relevant als het conversie-effect negatief is.
